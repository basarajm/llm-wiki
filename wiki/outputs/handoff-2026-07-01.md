- **KB금융 financial_products 오귀속**: `KB_원화예수금`·`KB_원화대출금`·`KB_신탁`·`KB_신용카드` 4종이
  자회사(국민은행·국민카드) 상품을 지주 `KB금융지주`에 잘못 귀속. issuer를 각 자회사로 재지정하고
  KB국민은행/KB국민카드 stub 생성 필요. (하나금융지주는 본문에 자회사 주체를 명시하는 방식으로 처리한 선례 참고.)
- **HS화성(002460)·현대제철(004020)**: 백필 시 원본 md가 감사보고서 첨부정정본이라
  「사업의 내용」·「신용평가」 섹션 부재 → products/ratings 0건. 향후 본문 사업보고서 재확보 후 백필 보강 필요.
- **`local_archive_status.py` 파일명≠회사명 오탐 주의(## 3-1)**: 아카이브 파일명이 회사의 과거명·약칭이거나
  wiki 회사명과 다른 경우, 스크립트는 파일명 기준으로 done을 판정해 이미 완료된 회사를 **영구히 pending으로
  오분류**한다. 확인된 사례(재인제스트 금지 — 이미 완료됨):
  - `APS이노베이션` → 실제 **디이엔티**(`wiki/companies/디이엔티.md`)
  - `CS` → 실제 **씨에스**(CS Corporation, `wiki/companies/씨에스.md`) — 기존 `CS홀딩스`와는 별개 회사
  - `DB` → 실제 **DB Inc.**(`wiki/companies/DB Inc..md`, DB그룹 지주회사)
  - `DGI` → 실제 **디지아이티엑스**(`wiki/companies/디지아이티엑스.md`)
  - `DH오토넥스` → 이미 `wiki/companies/DH오토웨어.md`(title: DH오토넥스)로 인제스트 완료(본 세션 이전).
  `--next` 결과에 위 5개 이름이 나오면 건너뛰고 다음 항목으로 채울 것. 새로운 사례 발견 시 이 목록에 추가.
- **딥 deferred(깨진 링크 허용 관례)**: 대기업집단 일부 그룹 페이지·다수 상장 자회사 stub 미생성.
  해당사 ingest 시 또는 일괄 lint로 해소.
- **국민연금 등 기관 주주**: 보고서에 5% 미만이면 미기재 → "미확인" 처리한 사례 다수.
- **경미한 사전 이슈(lint 대상, 이번 세션 확인만 하고 미수정)**:
  - `wiki/products/원사(THREE STAR).md` vs `원사_THREE_STAR.md` — 일신방직 원사 제품이 서로 다른 세션에서

  - `wiki/products/고밀도폴리에틸렌(HDPE).md`·`폴리프로필렌(PP).md` — 파일명에 괄호가 남아있어 마크다운
    링크 파싱 시 첫 `)`에서 끊길 위험(다른 파일들은 이미 `_`로 정규화함). 정규화 권장.
  - `wiki/companies/DB하이텍.md`의 `/shareholders/국민연금` 링크(실제 파일은 `국민연금공단.md`),

## 4. 환경·협업 주의

- **원격 협업**: 사용자가 Quartz 정적사이트(site/, gh-pages)도 origin에 푸시 중.
  매 배치 push 전 `git fetch && git rebase origin/main`로 공존(절대 force-push 금지).
- **읽기 전용**: `source_documents/`, `dart_pipeline/dart_md/` 원본은 절대 수정 금지.
- **줄바꿈 경고**: LF→CRLF git 경고는 무해(무시). 콘솔 한글 깨짐은 표시상 문제이며 파일은 정상 UTF-8.
- **명칭 매핑**: DART명≠위키명 사례 — 신한지주→신한금융지주, 삼성생명→삼성생명보험,
  POSCO홀딩스→포스코, 엘에스일렉트릭→LS일렉트릭, 삼성화재해상보험→삼성화재 등.
  불일치 시 `ingest_status.py KOSPI --mark "DART명"`.
