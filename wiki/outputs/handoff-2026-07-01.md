## 2. 재개 절차 (다른 PC)

### 사전 준비
1. 리포 clone 후 `origin/main` 최신화 (`git pull`).
2. **원본 보고서**: `dart_pipeline/dart_md/KOSPI/`·`KOSDAQ/`의 md 파일이 로컬에 있어야 함.
   `.gitignore`로 리포에서 제외됐을 수 있으니, 없으면 DART에서 재다운로드 필요
   (`dart_pipeline/` 도구 사용, [[dart-pipeline-overview]] 참조). `source_documents/`도 동일.
3. **DART API 키**는 환경변수로 주입(`dart_pipeline/config.py`가 env에서 읽음).
   키를 코드/깃에 하드코딩하지 말 것.

### 인제스트 1배치 루프 (3개사씩)
1. 다음 3개사 산출:
   ```
   python dart_pipeline/ingest_status.py KOSPI   # done/pending 갱신
   ```
   `compute()`의 pending을 stock_code순 정렬해 상위 3개를 `wiki/outputs/_ingest_next3.json`에 기록
   (각 항목: name·ticker·file(dart_md 경로)·fy). 위키노드명≠DART corp_name이면 수동 확인.
2. 3개사 각각 **general-purpose 서브에이전트**(백그라운드)로 Full 인제스트.
   각 에이전트는 **회사 전용 페이지만 작성**(아래 7종 스코프), 공유 노드는 **수정 금지**하고
   구조화 TODO(MARKET/GROUP/SHAREHOLDER/INDUSTRY/STUB)로 반환.
3. 세 에이전트 완료 후 **메인이 공유 노드 직접 적용**:
   - `wiki/markets/KOSPI.md` 상장기업 목록에 신규 3사 추가(가나다순 위치)
   - `wiki/groups/*` 신규/갱신 (그룹 소속 시)
   - `wiki/shareholders/*` 신규 (주요 주주, 특히 상장사·개인 대주주; company_page 양방향)
   - `wiki/industries/*` 주요 플레이어 목록 갱신(없으면 신규 생성)
4. 배치 마감:
   ```
   # wiki/log.md에 **Ingest(IG배치N)** 1블록 추가
   pwsh -NoProfile -File "engine\scripts\build-index.ps1"   # index 재생성
   python dart_pipeline/ingest_status.py KOSPI --mark "회사A,회사B,회사C"
   git add wiki/ dart_pipeline/
   git commit -m "ingest(IG): ... (N/833)"
   git fetch origin && git rebase origin/main && git push origin main
   ```

### Full 인제스트 7종 스코프(회사당)
1. `sources/<회사>_2025_사업보고서.md` (type: Annual Report, `resource:` 원본 dart_md 경로)
2. `companies/<회사>.md` (type: Company, is_stub:false) — frontmatter `ticker/market/market_page/industry/group`,
   재무 하이라이트 3개년, 사업개요·사업부문·주요주주·밸류체인·경영진·신용등급·리스크·# Citations
3. `segments/<회사>_<부문>.md` (type: Business Segment) — 부문별 매출비중
4. `value_chain/<회사>_밸류체인.md` (type: Value Chain)
5. `executives/<임원>.md` (type: Executive, company 역링크) — 핵심 경영진
6. `products/<상품>.md` (type: Product) — **점유율/순위/매출비중/대표성 명시 시에만**.
   일반명 충돌 시 `<회사>_상품명` 프리픽스. 지주회사는 자회사 상품을 지주에 귀속 금지.
7. `ratings/<회사> 신용등급.md` (type: Credit Rating) — 회사채/CP/기업신용(ICR) 등급 명시 시에만.
   금융사(증권·보험·은행·캐피탈)는 products 대신 `financial_products/`.
   + 식별 가능한 공급사·고객사·경쟁사·자회사는 `companies/`에 stub(is_stub:true).

### 양방향 링크 규칙 (CLAUDE.md 참조)
주주↔기업, 시장↔기업, 그룹↔기업, 임원↔기업, 공급/고객↔기업을 양방향 링크.
링크 경로는 번들 기준 절대경로(`/companies/...`).

## 4. 환경·협업 주의

- **원격 협업**: 사용자가 Quartz 정적사이트(site/, gh-pages)도 origin에 푸시 중.
  매 배치 push 전 `git fetch && git rebase origin/main`로 공존(절대 force-push 금지).
- **읽기 전용**: `source_documents/`, `dart_pipeline/dart_md/` 원본은 절대 수정 금지.
- **줄바꿈 경고**: LF→CRLF git 경고는 무해(무시). 콘솔 한글 깨짐은 표시상 문제이며 파일은 정상 UTF-8.
- **명칭 매핑**: DART명≠위키명 사례 — 신한지주→신한금융지주, 삼성생명→삼성생명보험,
  POSCO홀딩스→포스코, 엘에스일렉트릭→LS일렉트릭, 삼성화재해상보험→삼성화재 등.
  불일치 시 `ingest_status.py KOSPI --mark "DART명"`.
